Smart Crowd Monitoring & Safety System

Overview

The Smart Crowd Monitoring & Safety System is an AI-powered platform that uses computer vision to detect and track people in real time, providing live occupancy analytics, configurable safety alerts, restricted-zone monitoring, entry/exit counting, and detailed historical reporting through a modern web dashboard.

Technology Stack
  • Python & Flask
  • OpenCV
  • Ultralytics YOLOv8
  • ByteTrack (Supervision)
  • HTML5, CSS3, JavaScript
  • Bootstrap 5
  • Chart.js
Core Capabilities
  • Real-time person detection & tracking
  • Live occupancy & historical analytics
  • Configurable crowd threshold alerts
  • Restricted zone & line-crossing counting
  • Heatmap generation
  • CSV / JSON / TXT / PDF reporting
Responsible Use

This system was built for academic, research, and demonstration purposes. Any real-world deployment must comply with local privacy, surveillance, and data-protection laws.

Help & FAQ

Frequently Asked Questions

Quick answers to common operational questions.

  • Connect a webcam, CCTV/IP camera, or upload a video.
  • Click Start Monitoring.
  • The AI engine initializes automatically.
  • Live occupancy and analytics begin immediately.

Ensure the selected camera is connected and not being used by another application. Refresh the camera list and restart monitoring if the issue persists.

Verify that people are clearly visible in the camera frame. Poor lighting, extreme viewing angles, or low-resolution video may reduce detection accuracy.

Low FPS may occur due to high-resolution video, multiple active camera streams, or limited system resources and also depends on System Processor performane. Reducing the video resolution can improve performance.

Temporary counting variations may occur when people overlap or move quickly. Ensure the camera has a clear view of the monitored area and adjust the confidence threshold if required.

Verify that restricted zones are properly configured and saved. Alerts are generated only when a tracked person enters a defined restricted zone.

Check the camera URL, network connection, username, and password. Ensure the IP camera supports the configured streaming protocol.

Verify that the selected file is a supported video format and is not corrupted. Large video files may require additional processing time.

Confirm that monitoring is active and the backend service is running correctly. Refresh the dashboard or restart the monitoring session if updates stop unexpectedly.

Ensure the Flask backend server is running successfully and accessible. Verify that the frontend is connected to the correct backend service before starting monitoring.

SmartCrowd uses the YOLOv8 object detection model to detect people in every video frame. ByteTrack maintains unique IDs for detected individuals, enabling occupancy counting, tracking, and crowd analysis.

Alerts are generated whenever occupancy exceeds the configured threshold or when a restricted zone violation is detected.

Open the Reports page, choose the desired format, and click Generate Report. Generated reports can be downloaded directly.

Crowd Alert Threshold defines the maximum number of people allowed before the system generates a crowd alert.

Example:
  • Threshold = 20
  • Current Occupancy = 18 → No alert
  • Current Occupancy = 21 → Alert generated
Lower values trigger alerts earlier, while higher values allow larger crowds before an alert is raised.

Target Inference Rate specifies how many video frames are processed by the AI every second.

Higher values:
  • More accurate tracking
  • Smoother monitoring
  • Higher CPU/GPU usage
Lower values:
  • Lower resource consumption
  • Reduced power usage
  • Less responsive detection
A value between 10–15 FPS is suitable for most monitoring applications.

Alert Cooldown specifies the minimum time between two consecutive alerts.

Example:
  • Cooldown = 20 seconds
  • An alert occurs now.
  • No additional alert will be generated for the next 20 seconds.
This prevents repeated notifications when the same crowd condition persists.

Geometry Configuration allows you to define monitoring regions and counting lines directly on the video frame.

Using the Geometry Editor you can:
  • Create Restricted Zones
  • Draw a Counting Line
  • Configure Zone Alert Thresholds
  • Save the geometry for future monitoring sessions
All geometry is stored using the original video coordinates to ensure accurate detection regardless of screen size.

Restricted Zone Threshold defines how many people are allowed inside a configured restricted zone before an alert is generated.

Example:
  • Threshold = 5
  • People inside zone = 3 → Normal
  • People inside zone = 6 → Restricted Zone Alert
Each restricted zone can have its own threshold, allowing different safety limits for different monitored areas.

Camera Source Guide
🖥 Webcam

Uses your laptop's built-in camera. Best for testing and demonstrations.

📷 USB Camera

Uses an external USB webcam connected to your computer.

🌐 RTSP Stream

Connects directly to CCTV cameras and NVR systems. 

rtsp://username:password@192.168.1.20:554/stream
📡 IP / HTTP Camera

Connects using HTTP or MJPEG streams. 

http://192.168.1.25/video
🎥 Uploaded Video

Analyze an MP4, AVI, MOV or WebM video without using a live camera.

Tip
Choose Webcam if you're using your laptop camera.
